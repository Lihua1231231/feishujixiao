# 绩效初评分布校准页设计

## 背景

CEO 的诉求有两层，而且都只针对 **绩效初评**：

- 先校正评分人的打分尺度，修正“某些初评人整体打分偏高或偏低”的问题
- 再按部门做强制正态分布，方便部门内排名和后续校准

这不是 360 环评的问题。  
**360 环评不进入这套正态分布流程。**

当前系统已经有：

- 直属上级初评加权分
- 部门与组织结构信息
- 绩效校准页
- 后台核验和导出

但还没有一个独立页面，能把“原始初评分”“评分人校正后结果”“部门强制分布结果”放在一起分析、模拟、确认和回退。

## 目标

- 新增一个独立页面：`绩效初评分布校准`
- 页面只处理 **绩效初评**，不混入 360 环评
- 清楚展示三套口径：
  - 原始初评分
  - 评分人尺度校正后的连续分值
  - 部门强制正态分布后的连续分值
- 同时给出参考星级，但连续分值优先用于排名和判断
- 提供明确的应用与回退能力

## 非目标

- 不修改原始初评记录
- 不修改 360 环评记录
- 不在本轮把标准化结果直接写回 `SupervisorEval`
- 不替换现有绩效校准页的主体流程
- 不采用难解释的 z-score 口径

## 核心原则

### 1. 只处理绩效初评

本页只分析和模拟 **绩效初评加权分**。  
所有 360 环评数据都排除在这套功能之外。

### 2. 先校正评分人，再做部门分布

整个流程固定为两段：

1. 先按初评人维度，识别并校正评分尺度偏差
2. 再在每个部门内部，基于校正后的结果做强制正态分布模拟

### 3. 原始分永远保留

系统中必须并存三套值：

- 原始初评分
- 评分人校正后的连续分值
- 部门强制分布后的连续分值

任何“应用标准化结果”的操作，都不能覆盖原始初评分。

### 4. 应用必须可回退

应用后：

- 页面顶部持续显示“当前正在使用标准化口径”
- 必须有清晰的回退按钮
- 回退后立即恢复到原始口径

## 目标分布

部门强制正态分布统一使用当前绩效体系已有的目标分布：

- `5星 ≤ 10%`
- `4星 ≤ 20%`
- `3星 ≥ 50%`
- `2星 ≤ 15%`
- `1星 ≤ 5%`

极小部门不强行凑满每一档，采用“**近似满足目标分布**”的处理方式：

- 优先保证 `3星` 是主体
- 剩余名额按排序结果分配
- 页面明确标注“该部门因人数过少，采用近似分布”

## 数据口径

### 基础分

基础分使用当前系统里的 **直属上级初评加权分**。

### 对象范围

分析对象为当前活动周期中，参与绩效初评与后续校准的人。

### 部门口径

部门分布与部门排名都以当前用户所属部门为准。

## 标准化算法

### 第一段：评分人尺度校正

目的：

- 修正某个初评人整体偏高或偏低的打分倾向

推荐做法：

- 以每个初评人为单位，收集其当前所有有效初评对象的原始加权分
- 计算该初评人的评分分布
- 将其对象在该评分人内部排序
- 基于统一口径生成一套“评分人校正后的连续分值”

这一层的结果不直接用于组织排名，只用于消解评分人尺度差异。

### 第二段：部门强制正态分布

目的：

- 让部门内的结果更适合比较、排名和后续组织校准

做法：

- 以部门为单位，读取该部门成员的“评分人校正后连续分值”
- 在部门内重新排序
- 按目标分布分配名额，生成“部门强制分布后的连续分值”和“参考星级”

### 最终输出

每个人都需要同时展示：

- 原始初评分
- 评分人校正后连续分值
- 部门强制分布后连续分值
- 参考星级

## 页面结构

页面名：

- `绩效初评分布校准`

采用驾驶舱式布局。

### 顶部摘要带

显示：

- 当前是否启用标准化口径
- 异常评分人数量
- 当前正在看的部门
- 分布偏离最严重的部门数量
- 最近应用时间
- `回退到原始分` 按钮（仅已应用时显示）

### 第一块：评分人尺度校正

#### 左侧：评分人分布图

每个评分人一张分布卡或可切换图表，展示：

- 原始打分分布
- 平均分偏移
- 偏高 / 偏低标签
- 校正后整体变化

#### 右侧：评分人影响明细

展示：

- 被该评分人评价的人
- 原始初评分
- 校正后连续分值
- 变化幅度

这部分回答：

- 谁的评分尺度有问题
- 校正后影响了哪些人

### 第二块：部门强制正态分布模拟

#### 上层主图：部门分布对比图

选中一个部门后，展示三联柱状图：

- 原始分布
- 评分人校正后分布
- 强制正态分布后分布

横轴：

- `1星 ~ 5星`

纵轴：

- 人数

这张图回答：

- 这个部门原来多偏
- 评分人校正能拉回多少
- 强制分布后最终会变成什么样

#### 下层主图：部门内排名变化图

采用排名变化线图（bump chart）：

- 左列：原始排名
- 中列：评分人校正后排名
- 右列：强制分布后排名

鼠标悬停时展示：

- 姓名
- 原始初评分
- 校正后连续分值
- 强制分布后连续分值
- 参考星级

这张图回答：

- 谁升了
- 谁降了
- 排名变化最大的几个人是谁

### 第三块：应用 / 回退

这是一个单独的重操作区。

包含：

- 当前应用状态
- 本次应用会影响哪些页面和统计
- 双重确认复选框
- `应用标准化结果` 按钮
- `回退到原始口径` 按钮

## 应用范围

应用标准化结果后，只影响：

- 这个新页面里的结果展示
- 绩效校准页里的分布、排名、参考判断
- 后台核验与导出里和排名相关的统计

不影响：

- 原始 `SupervisorEval` 记录
- 原始 360 环评记录

## 权限

建议允许访问的人：

- `ADMIN`
- `吴承霖`
- `邱翔`
- `禹聪琪`
- 其他被明确授予查看或操作权限的人

建议区分：

- 只读查看者
- 可执行应用 / 回退的人

## 回退

回退后：

- 相关页面重新按原始初评分显示
- 新页面继续保留历史快照，方便对比
- 不需要重新计算原始数据

## 验收标准

完成后应满足：

1. 页面只处理绩效初评，不出现 360 相关分析
2. 能清楚看出哪个初评人打分偏高或偏低
3. 能按部门查看原始分布、校正后分布、强制分布后的变化
4. 能看到部门内排名变化
5. 能应用标准化结果，并一键回退
6. 应用与回退都不改原始初评记录
