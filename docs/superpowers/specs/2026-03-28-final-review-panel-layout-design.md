# 终评工作台面板式布局设计

## 背景

上一轮终评工作台已经完成了业务能力和权限收口，但第二页 `非主管员工终评` 仍然不符合真实使用习惯：

- 页面像“信息展示台”，不像“逐个处理员工”的工作台
- 图表、名单、详情同时抢注意力，主次不清
- 右侧虽然有详情面板，但左侧仍然堆了过多分析块，导致处理一个人时不够专注
- 用户已经明确表达希望采用更典型的绩效系统做法：
  - 左侧选人
  - 右侧只处理当前这一个人
  - 打完之后由人自己决定是否切到下一位

这次设计不改变最初确定的三标签结构，也不改变权限和业务规则；只重做终评工作台的页面组织方式，特别是第二页和第三页的空间分配与交互节奏。

## 目标

- 把第二页真正改成“面板型终评处理台”
- 左侧承担“选人和切换队列”，右侧承担“当前单人处理”
- 减少持续常驻的大盘信息，避免页面再次变成大杂烩
- 让确认动作完成后，状态变化明显，但不强制自动切人
- 第三页主管层终评沿用同一结构语言，降低学习成本

## 非目标

- 不改变三标签结构：`原则`、`非主管员工终评`、`主管层双人终评`
- 不改变当前的终评权限模型
- 不改变普通员工终评的 54 人默认名单规则
- 不改变参考星级、官方星级、最终确认等已有业务口径
- 不重做第一阶段原则页的业务内容

## 核心设计结论

### 1. 第二页采用 A 方案

页面按 `左侧队列 + 右侧单人工作区` 重构：

- 左侧固定为处理队列，不再承载复杂图表
- 右侧永远只围绕当前选中的一个员工展开
- 用户确认后仍停留在当前人，不自动切到下一个
- 当前人会立即出现明显状态提示，告诉用户“已确认”，并提示可手动切换下一位

### 2. 第二页顶部的大盘信息改成抽屉

`公司分布 / 团队分布 / 分数带` 不再常驻在页面主视野中。

改为：

- 页面顶部保留一个轻量入口：`查看整体分布`
- 点击后展开抽屉，显示当前整体分布和必要解释
- 默认收起，让页面主体优先服务“选一个人、处理一个人”

### 3. 右侧采用单列长页面，不做标签切换

右侧单人工作区不使用 `决策 / 证据 / 留痕` 标签页切换，而是按自然阅读顺序单列展开。

原因：

- 更符合常见绩效系统的处理方式
- 不会让用户额外记忆“信息藏在哪个标签里”
- 对第一次使用的人最直观

## 标签 2：非主管员工终评

## 页面结构

### 顶部：轻量头部

只保留一行说明和一个入口动作：

- 页面说明：这一页用于逐个处理普通员工终评
- 抽屉入口：`查看整体分布`

不再保留常驻的多张统计卡和双图并排布局。

### 左侧：处理队列

左侧固定宽度，职责只有一个：帮用户找到当前要处理的人。

包含三种切换视图：

- `待拍板`
- `有分歧`
- `全部员工`

默认进入 `待拍板`。

每个队列项只显示最必要的信息：

- 姓名
- 部门
- 当前状态
- 参考星级 / 官方星级
- 简短风险标签（如有）

### 队列交互规则

- 点击某个人，右侧切换到该员工
- 不自动滚动到其他区块
- 当前选中项有明确高亮
- 已确认的人仍保留在队列中，方便回看

### 右侧：单人工作区

右侧固定为当前员工的单列长页面，顺序如下：

1. `当前结论`
2. `证据摘要`
3. `初评评语摘要`
4. `意见汇总`
5. `最终确认`
6. `过程留痕`

## 右侧详细分区

### 当前结论

放在最顶端，第一眼就能看懂：

- 姓名
- 部门 / 岗位
- 参考星级
- 当前官方星级
- 当前状态标签

状态标签至少包含：

- `待拍板`
- `已确认`
- `有分歧`

当当前人刚被确认后：

- 状态立即切为 `已确认`
- 旁边增加轻提示，例如：`已确认，可切换下一位`
- 页面不自动跳人

### 证据摘要

这一块只保留做决策必须的信息：

- 初评加权分
- 360 均分
- 风险信号
- 初评人

目的不是展示全部上下文，而是支持快速判断。

### 初评评语摘要

初评评语摘要默认折叠展示：

- 默认显示有限行数
- 提供 `展开全文 / 收起全文`

这块保留在证据摘要之后，避免一开始就被长文本压住。

### 意见汇总

普通员工终评中，这一块只展示 3 位具名拍板人：

- 吴承霖
- 邱翔
- 禹聪琪

向金涛和陈琼仍然可以查看页面，但不在这个具名意见区占位，也不出现在计数口径里。

意见汇总分两层：

- 汇总层：已处理人数、待处理人数、改星人数
- 具名层：只在允许查看具名细节时展开

### 最终确认

保持在右侧中下部，接近自然操作终点：

- 选择官方星级
- 填写确认理由
- 点击确认

确认完成后：

- 不跳转
- 不自动切换下一个人
- 只更新当前人的状态和标签

### 过程留痕

始终放在最底部，不参与主决策视线。

保留：

- 官方确认理由
- 最后确认时间
- 最后确认人
- 具名更新时间（按权限）

## 整体分布抽屉

第二页顶部的大盘分析统一进入一个抽屉。

抽屉中保留：

- 星级分布
- 分数带图
- 必要的队列解释

抽屉的职责是：

- 在需要时帮助管理层回到整体分布
- 平时不占据页面主空间

抽屉默认关闭。

## 标签 3：主管层双人终评

主管层沿用同一信息架构，但内容适配双人终评场景。

### 左侧

左侧同样使用队列化结构，至少保留：

- `待拍板`
- `待双人齐备`
- `全部主管`

### 右侧

右侧同样是单人工作区，但中段内容改为主管层所需：

1. 当前结论
2. 双人意见摘要
3. 详细双人问卷（按权限）
4. 最终确认
5. 过程留痕

### 主管层交互约束

- 问卷编辑权仍只属于两位主管层填写人
- 最终确认权仍只属于最终确认人
- 非对应角色看到的是摘要和结果，而不是可编辑问卷

## 标签 1：原则

`原则` 页不做结构性重写，只保持与新布局兼容。

要求：

- 继续保留简洁 briefing 角色
- 继续作为三标签的入口页
- 文案保持简短，不扩张内容

## 视觉方向

页面风格继续沿用当前终评工作台的温和中性色，不改整体品牌方向。

但布局要更接近“处理台”而不是“总览台”：

- 左侧更像队列栏
- 右侧更像工作面板
- 抽屉承担整体分析
- 页面中心不再同时出现多组互相竞争的大卡片

## 成功标准

如果这次重排成功，第二页应当满足：

- 用户进入后，第一直觉是“先从左边选一个人”
- 用户处理过程中，不会被持续常驻的大盘图打断
- 用户能清楚知道当前是不是已经确认
- 用户在确认之后不会被系统强制跳转
- 用户如果想看整体分布，能在同一页方便展开抽屉查看

第三页应当满足：

- 与第二页结构一致，学习成本低
- 双人终评与最终拍板的职责清楚分层

## 实现影响

这次属于布局和交互重排，不引入新的业务模型。

预期改动集中在：

- 第二页普通员工 cockpit 组件
- 第三页主管层 cockpit 组件
- 对应 detail panel 的区块组织
- 少量状态文案与提示标签

不涉及：

- 新接口
- 新数据库字段
- 新权限模型
